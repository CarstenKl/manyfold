## Overview

This repository contains a single Proxmox VE LXC container deployment script for [Manyfold](https://github.com/manyfold3d/manyfold), an open-source 3D model management web application. The script follows the [community-scripts/ProxmoxVE](https://github.com/community-scripts/ProxmoxVE) framework conventions.

The actual Manyfold application source is not in this repo — it is fetched from GitHub at deploy time.

## Script Structure

`manyfold.sh` has two main sections:

1. **Header/defaults** (lines 1–20): Sources the community-scripts build framework, sets default container specs (4 CPU, 4 GB RAM, 15 GB disk, Debian 13, unprivileged).

2. **`update_script()` function** (lines 22–100): Called when updating an existing installation. Performs the full lifecycle:
   - Stops systemd services
   - Backs up `storage/`, `tmp/`, `credentials.yml.enc`, and `master.key` before wiping the app directory
   - Fetches the latest release tarball from `manyfold3d/manyfold` via `fetch_and_deploy_gh_release`
   - Reads Ruby version from `.ruby-version` and Yarn version from `package.json`
   - Installs dependencies as the `manyfold` user using rbenv-managed Ruby + Yarn via corepack
   - Runs `bin/rails db:migrate` and `bin/rails assets:precompile`
   - Restores backed-up data
   - Re-exports Foreman → systemd services and restarts them

3. **Initial install** (lines 102–109): Delegates to `start` / `build_container` / `description` from the framework.

## Key Paths (inside the container)

| Path | Purpose |
|------|---------|
| `/opt/manyfold/app/` | Manyfold application root |
| `/opt/manyfold/.env` | Environment variables (including `APP_VERSION`) |
| `/home/manyfold/.rbenv/` | rbenv Ruby installation for `manyfold` user |
| `/etc/systemd/system/manyfold-*.service` | Systemd units generated by Foreman |

## Systemd Services

The app runs as four systemd units managed under `manyfold.target`:
- `manyfold-rails.1` — Puma/Rails web server
- `manyfold-default_worker.1` — Sidekiq default queue
- `manyfold-performance_worker.1` — Sidekiq performance queue

## Framework Conventions

Helper functions (`msg_info`, `msg_ok`, `msg_error`, `check_for_gh_release`, `fetch_and_deploy_gh_release`, `setup_nodejs`, `setup_ruby`, `get_latest_github_release`, `$STD`) all come from the sourced community-scripts framework and are not defined locally.

When modifying this script, follow the patterns used by other scripts in the [community-scripts/ProxmoxVE](https://github.com/community-scripts/ProxmoxVE) repository to stay compatible with framework updates.
